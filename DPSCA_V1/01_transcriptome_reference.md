# PantheonOS Skill: Transcriptome Reference Construction & QC

This comprehensive guide details the standardized workflows for transcriptome reference construction, quality control, and redundancy removal based on the provided sequencing data types. It covers NGS short-read data, PacBio Iso-Seq long-read data, and combined multi-omics hybrid error correction.

> **Global Output Note:** All outputs generated by the workflows described in this document must be saved in the `01_transcriptome_reference` folder within the current working directory.

---

## 1. NGS Data Only: De Novo Transcriptome Assembly

When only Next-Generation Sequencing (NGS) short-read data is provided, the standard workflow consists of quality control, de novo assembly, extraction of the longest transcripts, and sequence redundancy removal.

### Core Workflow

```bash
# Step 1: Quality control using fastp with default parameters
fastp -i input_R1.fastq.gz -I input_R2.fastq.gz \
      -o clean_R1.fastq.gz -O clean_R2.fastq.gz

# Step 2: De novo transcriptome assembly using Trinity with default parameters
Trinity --seqType fq \
        --left clean_R1.fastq.gz \
        --right clean_R2.fastq.gz \
        --output trinity_out_dir

# Step 3: Extract the longest isoform per gene using the Trinity utility script
get_longest_isoform_seq_per_trinity_gene.pl trinity_out_dir/Trinity.fasta > longest_isoforms.fasta

# Step 4: Remove redundant transcripts to generate the final reference using cd-hit-est
cd-hit-est -i longest_isoforms.fasta -o transcriptome.fasta
```
>[!TIP]
Note on CD-HIT: For nucleotide sequences (transcripts), the cd-hit-est command from the cd-hit suite must be used instead of the standard cd-hit command, which is designed for protein sequences.

## 2. PacBio Data Only: Iso-Seq Pipeline
When only Third-Generation PacBio data is provided, the workflow focuses on the extraction, clustering, collapsing of full-length transcripts, and a final redundancy removal step.

### Core Workflow
```bash
# Step 1: Demultiplex samples using lima with default parameters
lima input.ccs.bam barcodes.fasta demux.bam

# Step 2: Extract Full-Length Non-Chimeric (FLNC) sequences using isoseq refine with default parameters
isoseq refine demux.bam primers.fasta flnc.bam

# Step 3: Cluster FLNC sequences to generate consensus transcripts using isoseq cluster with default parameters
isoseq cluster flnc.bam clustered.bam

# Step 4: Merge and deduplicate sequences using isoseq collapse with default parameters
isoseq collapse clustered.bam collapsed.gff

# Step 5: Remove sequence redundancy to generate the final reference using cd-hit-est
cd-hit-est -i collapsed.fasta -o transcriptome.fasta
```
## 3. NGS & PacBio Data Provided: Hybrid Error Correction
Third-generation sequencing reads often contain higher random insertion/deletion error rates. When users provide both highly accurate NGS short reads and long PacBio reads, the long reads should be corrected using the NGS data prior to final redundancy removal.

### Core Workflow
```bash
# Step 1: Correct PacBio long reads based on NGS short reads using LORDEC with default parameters
# -2: Input paired-end NGS reads (interleaved or comma-separated list)
# -i: Input uncorrected PacBio long reads
# -o: Output corrected PacBio long reads
lordec-correct -2 ngs_short_reads.fastq.gz -i pacbio_long_reads.fasta -o corrected_pacbio.fasta

# Step 2: Remove sequence redundancy to generate the final reference using cd-hit-est
cd-hit-est -i corrected_pacbio.fasta -o transcriptome.fasta
```

## Recommended Decision Framework
PantheonOS should automatically trigger the following logic pathways based on the input data types detected:

> IF [NGS Only]: Execute fastp -> Trinity -> get_longest_isoform_seq_per_trinity_gene.pl -> cd-hit-est (Outputs transcriptome.fasta)

> IF [PacBio Only]: Execute lima -> isoseq refine -> isoseq cluster -> isoseq collapse -> cd-hit-est (Outputs transcriptome.fasta)

> IF [NGS + PacBio]: Execute LORDEC for hybrid error correction -> [Optional downstream assembly] -> cd-hit-est (Outputs transcriptome.fasta)
